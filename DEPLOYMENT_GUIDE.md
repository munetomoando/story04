# ごはんマップ / ごはんナビ デプロイ・運用ガイド

このアプリは、任意の街・エリアを対象にした飲食店おすすめアプリです。
協調フィルタリングによる推薦機能を備え、環境変数でアプリ名を設定するだけで
「〇〇ごはんマップ」「〇〇ごはんナビ」として展開できます。

---

## 必要なもの

- **Git** でこのリポジトリをクローンできること
- **Docker** がインストールされていること（ローカルビルドの場合）
- デプロイ先のアカウント（Fly.io、Render.com、Railway 等）

---

## 環境変数一覧

| 変数名 | 必須 | 説明 | 例 |
|--------|------|------|-----|
| `APP_NAME` | 推奨 | アプリの表示名（ヘッダー、タイトル等） | `神保町ごはんナビ` |
| `ADMIN_USERNAME` | 必須 | 管理者ログインユーザー名 | `admin` |
| `ADMIN_PASSWORD_HASH` | 必須 | 管理者パスワードの bcrypt ハッシュ | `$2b$12$...` |
| `SESSION_SECRET_KEY` | 推奨 | セッション暗号化キー（未設定時は自動生成・ファイル永続化） | 64文字の16進数 |
| `INVITE_CODE` | 任意 | ユーザー登録に必要な招待コード（未設定で自由登録） | `welcome2025` |
| `SEED_DATA` | 任意 | `true` で初期サンプルデータを投入（デフォルト: `true`） | `false` |
| `DATA_DIR` | 任意 | データ保存先ディレクトリ（デフォルト: `/data`） | `/data` |
| `PORT` | 任意 | サーバーポート（デフォルト: `8080`） | `8080` |

### 管理者パスワードハッシュの生成方法

```bash
python3 -c "import bcrypt; print(bcrypt.hashpw('あなたのパスワード'.encode(), bcrypt.gensalt()).decode())"
```

出力される `$2b$12$...` の文字列を `ADMIN_PASSWORD_HASH` に設定してください。

## Fly.io へのデプロイ

### 初回セットアップ

```bash
# 1. リポジトリをクローン
git clone <リポジトリURL>
cd lunchmap

# 2. Fly.io にログイン
fly auth login

# 3. 新しいアプリを作成（アプリ名は一意にする）
fly launch --name jimbocho-gohan-navi --region nrt --no-deploy

# 4. 永続ボリュームを作成（データ保存用）
fly volumes create data --size 1 --region nrt

# 5. 環境変数を設定
fly secrets set \
  APP_NAME="神保町ごはんナビ" \
  ADMIN_USERNAME="admin" \
  ADMIN_PASSWORD_HASH='$2b$12$ここにハッシュを貼り付け' \
  SEED_DATA="false"

# 6. デプロイ
fly deploy
```

### fly.toml の編集

`fly.toml` のアプリ名を変更してください：

```toml
app = 'jimbocho-gohan-navi'  # あなたのアプリ名に変更
```

`[mounts]` セクションが以下のようになっていることを確認：

```toml
[mounts]
  source = "data"
  destination = "/data"
```

### 招待コードを設定する場合

```bash
fly secrets set INVITE_CODE="your-invite-code"
```

これを設定すると、ユーザー登録時に招待コードの入力が必要になります。

## Render.com へのデプロイ

### セットアップ

1. [Render.com](https://render.com) にログイン
2. 「New Web Service」を選択
3. このリポジトリを接続
4. 以下を設定：

| 設定項目 | 値 |
|----------|-----|
| Environment | Docker |
| Region | Singapore（またはお好みのリージョン） |
| Instance Type | Free または Starter |

5. 「Environment Variables」に以下を追加：

```
APP_NAME=渋谷ごはんマップ
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=$2b$12$ここにハッシュ
SEED_DATA=false
DATA_DIR=/data
```

6. 「Disk」を追加（データ永続化）：
   - Name: `data`
   - Mount Path: `/data`
   - Size: 1 GB

7. 「Create Web Service」をクリック

### 注意事項

- Render の無料プランではディスクが使えないため、Starter 以上が必要です
- `Dockerfile` の `CMD` でポートを `PORT` 環境変数から読み取るように設定済みです

## Railway へのデプロイ

1. [Railway](https://railway.app) にログイン
2. 「New Project」→「Deploy from GitHub repo」
3. このリポジトリを選択
4. 「Variables」タブで環境変数を設定（Fly.io と同じ変数）
5. 「Settings」で PORT を `8080` に設定
6. Volume を追加して `/data` にマウント

## デプロイ後の初期設定

### 1. 管理者ログイン

ブラウザで `https://あなたのアプリURL/admin/login` にアクセスし、
設定した `ADMIN_USERNAME` と パスワード でログインします。

### 2. 店舗の登録

1. 管理画面「店舗管理」ページを開く
2. 店舗追加リクエストがあれば承認、または直接追加
3. 各店舗に以下を設定：
   - **位置情報**（緯度・経度）— マップ表示に必要
   - **ジャンル** — 分類に使用
   - **写真** — ユーザーの評価画面に表示

### 3. ジャンルの設定

管理画面「ジャンル管理」で、その街に合ったジャンルを追加・削除・並び替えできます。
初回起動時にデフォルトの15ジャンルが登録されます。

### 4. ユーザーの招待

- `INVITE_CODE` を設定している場合：招待コードをユーザーに伝えてください
- 設定していない場合：誰でも登録ページからアカウントを作成できます

### 5. グループの作成（任意）

管理画面「グループ管理」で、部署やクラスなどのグループを作成できます。
ユーザーはグループコードを入力して所属を設定します。

## 日常の運用

### バックアップ

管理画面「ダッシュボード」最下部の「バックアップをダウンロード」ボタンから
ZIP ファイルをダウンロードできます。定期的にダウンロードしてローカルに保管してください。

ファイル名: `lunchmap_backup_YYYYMMDD.zip`

内容:
- データベース（ユーザー、評価、口コミ、設定すべて）
- 店舗画像
- セッション秘密鍵

### 復元

同じダッシュボードの「バックアップからの復元」で ZIP をアップロードすると
データを復元できます。復元前に整合性チェックが行われます。

### パスワードリセット

ユーザーがパスワードを忘れた場合：
1. 管理者がユーザー管理ページで「PW」ボタンを押す
2. 表示された仮パスワードを対面でユーザーに伝える
3. ユーザーがログイン後「パスワード変更」から自分のパスワードに変更

### 口コミの管理

- 通報された口コミはダッシュボードと口コミ管理ページに表示されます
- 削除した口コミは「削除済み」タブからアーカイブを確認・復元できます

### ユーザーの管理

ユーザーの状態遷移: `active` → `warned`（警告）→ `banned`（凍結）→ 削除

自動クリーンアップ:
- 登録後7日以内にログインがないアカウントは自動削除
- 最終ログインから2年以上経過したアカウントは inactive → 30日後に自動削除

## 複数の街での運用

同じコードベースから、異なる街のアプリを独立してデプロイできます。

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ 神保町ごはんナビ │  │ 渋谷ごはんマップ │  │ 下北沢ごはんナビ │
│ Fly.io           │  │ Render.com       │  │ Railway          │
│ APP_NAME=神保町… │  │ APP_NAME=渋谷…   │  │ APP_NAME=下北沢…│
│ 管理者: Aさん    │  │ 管理者: Bさん    │  │ 管理者: Cさん    │
└─────────────────┘  └─────────────────┘  └─────────────────┘
         │                   │                    │
         └───── 同じコード（Git リポジトリ）──────┘
```

各デプロイは完全に独立しています：
- データベースは別々（各プラットフォームの永続ストレージ）
- ユーザーは別々（街ごとに登録）
- 管理者は別々（街ごとに `ADMIN_PASSWORD_HASH` を設定）
- 店舗データは別々（管理者が街ごとに登録）

### 新しい街を追加する手順

1. デプロイ先のプラットフォームを選ぶ
2. 環境変数を設定（`APP_NAME`, `ADMIN_PASSWORD_HASH`, `SEED_DATA=false`）
3. デプロイ
4. 管理者ログインして店舗を登録
5. ユーザーを招待して運用開始

## ログインページのイラスト変更

デフォルトでは `static/images/suidobashi.png` が表示されます。
街に合った画像に差し替えるには：

1. 画像ファイルを `static/images/` に配置
2. `templates/index.html` の `<img src="/static/images/suidobashi.png"` を変更

または、環境変数で切り替える場合は `templates/index.html` を編集してください。

## トラブルシューティング

### アプリが起動しない

- `ADMIN_PASSWORD_HASH` が正しい bcrypt ハッシュか確認（`$2b$12$` で始まる）
- `DATA_DIR` のボリュームがマウントされているか確認
- ログを確認: `fly logs`（Fly.io）、ダッシュボードのログ（Render/Railway）

### 管理者ログインできない

- `ADMIN_USERNAME` と `ADMIN_PASSWORD_HASH` が正しく設定されているか確認
- ハッシュにシングルクォートが含まれる場合、シェルのエスケープに注意

### 推薦が表示されない

- 最低2人以上のユーザーが評価を完了している必要があります
- 類似度閾値（0.3）を超えるユーザーペアが必要です
- 評価対象の店舗が十分に登録されているか確認

### ディスク容量不足

- ダッシュボードからバックアップをダウンロード
- 不要な店舗画像を削除
- Fly.io: `fly volumes extend <vol_id> --size 2` で容量拡張

---

## セキュリティに関する注意

- `ADMIN_PASSWORD_HASH` は必ず bcrypt ハッシュを設定してください（平文不可）
- `SESSION_SECRET_KEY` を設定しない場合、自動生成されファイルに永続化されます
- HTTPS は Fly.io/Render/Railway で自動的に有効になります
- 全 POST エンドポイントは CSRF トークンで保護されています
- パスワードは bcrypt でハッシュ化されて保存されます
