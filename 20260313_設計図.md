# 神田美崎町ランチマップ — 設計図

**作成日**: 2026-03-13
**アプリ概要**: 協調フィルタリングによる飲食店レコメンドWebアプリ

---

## 1. 全体アーキテクチャ

```
┌─────────────┐     HTTPS      ┌──────────────────────────┐
│  スマートフォン │ ◄──────────► │  FastAPI (Uvicorn)        │
│  ブラウザ      │              │  port 8080                │
└─────────────┘              │                          │
                              │  ┌──────────────────┐    │
                              │  │ Jinja2 Templates  │    │
                              │  └──────────────────┘    │
                              │  ┌──────────────────┐    │
                              │  │ recommendation.py │    │
                              │  │ (協調フィルタリング)  │    │
                              │  └──────────────────┘    │
                              │           ↕              │
                              │  ┌──────────────────────┐│
                              │  │ CSV Files (/data)     ││
                              │  │ users / objects /     ││
                              │  │ ratings /             ││
                              │  │ recommendations /     ││
                              │  │ object_requests       ││
                              │  └──────────────────────┘│
                              └──────────────────────────┘
                                        ↑
                              ┌──────────────────┐
                              │ Fly.io (nrt)      │
                              │ Docker Container  │
                              │ Volume: /data     │
                              └──────────────────┘
```

**技術スタック**:
- **バックエンド**: FastAPI + Uvicorn
- **テンプレート**: Jinja2
- **データ永続化**: CSV ファイル（Fly.io Volume `/data` にマウント）
- **推薦エンジン**: scikit-learn (cosine_similarity), NumPy, Pandas
- **可視化**: Matplotlib, Seaborn (Base64 PNG埋め込み)
- **認証**: bcrypt パスワードハッシュ + Starlette SessionMiddleware
- **デプロイ**: Docker + Fly.io (東京リージョン nrt)
- **CI/CD**: GitHub Actions → `flyctl deploy`

## 5. 協調フィルタリングアルゴリズム

### 5.1 処理フロー（変更なし）

```
ratings.csv
    │
    ▼
┌───────────────────────────────┐
│ (1) ユーザー×店舗 評価行列作成   │  pivot_table (user_id × object_id)
│     (user_object_matrix)       │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ (2) Zスコア標準化               │  各ユーザーの評価を
│     (user_zscore_matrix)       │  z = (rating - mean) / std で正規化
│                               │  ※ std < 0.1 の場合は全て0
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ (3) コサイン類似度計算           │  sklearn cosine_similarity
│     (user_similarity)          │  Zスコア行列からユーザー間類似度を算出
│                               │  値域: 0〜1
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ (4) 推薦スコア算出              │  各ユーザーの未評価店舗について:
│                               │  - 類似度 ≥ 0.3 のユーザーを抽出
│                               │  - 類似ユーザーのZスコアを類似度で加重平均
│                               │  rec = Σ(z_score × sim) / Σ(sim)
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│ (5) 星レーティング変換           │  score ≥ 0.75 → ★★★
│                               │  score ≥ 0.50 → ★★
│                               │  score ≥ 0.25 → ★
│                               │  score < 0.25 → 表示なし
└───────────────────────────────┘
```

### 5.2 推薦計算のキャッシュ化（新規）

```
                  ┌────────────────────────────┐
                  │ 評価送信 (POST /submit_ratings) │
                  └──────────────┬─────────────┘
                                 │
                                 ▼
                  ┌────────────────────────────┐
                  │ recommend_for_all_users()   │
                  │ 全ユーザー分を一括計算         │
                  └──────────────┬─────────────┘
                                 │
                      ┌──────────┴──────────┐
                      ▼                     ▼
            ┌──────────────┐     ┌──────────────────┐
            │ メモリキャッシュ │     │ recommendations   │
            │ (グローバル変数) │     │ .csv に保存        │
            └──────────────┘     └──────────────────┘
                      │
                      ▼
            ┌──────────────────────────────┐
            │ GET /recommendations          │
            │ キャッシュから読み込み（再計算なし）│
            └──────────────────────────────┘
```

**計算タイミング**:
- 評価送信時（`POST /submit_ratings`）にのみ全ユーザー分を再計算
- 結果を `recommendations.csv` に保存 + メモリキャッシュに保持
- 閲覧時（`GET /recommendations`）はキャッシュを返すだけ
- サーバー再起動時は `recommendations.csv` から復元

### 5.3 主要関数 (recommendation.py)

| 関数 | 役割 |
|---|---|
| `compute_standardized_rating(series)` | ユーザー単位でZスコア計算 |
| `update_user_similarity_from_csv()` | CSV読込→評価行列→Zスコア行列→類似度行列をグローバル変数に格納 |
| `recommend_for_all_users(threshold=0.3)` | 全ユーザーの推薦スコアをDataFrameで返却 |
| `get_recommendations_for_user(username)` | 特定ユーザーの推薦結果を辞書で返却 (score ≥ 0.25) |
| `categorize_recommendation(score)` | スコアを星マークに変換 |
| `get_username(user_id)` | ユーザーID→ユーザー名を返却 |

### 5.4 主要パラメータ

| パラメータ | 値 | 説明 |
|---|---|---|
| 類似度閾値 | 0.3 | 推薦に使う類似ユーザーの最低コサイン類似度 |
| 表示閾値 | 0.25 | ユーザーに表示する推薦スコアの下限 |
| Zスコア最低std | 0.1 | これ未満は全評価を0として扱う |

## 6. 評価UI — カード形式（改修）

### 6.1 画面デザイン

```
┌─────────────────────────┐
│  ランチマップ    ☰ メニュー │  ← ヘッダー（固定）
├─────────────────────────┤
│                         │
│       3 / 20            │  ← 進捗（現在/全体）
│                         │
│  ┌─────────────────────┐│
│  │                     ││
│  │     🍜 吉野家        ││  ← 店舗名
│  │                     ││
│  │  ┌──┐┌──┐┌──┐┌──┐┌──┐││
│  │  │1 ││2 ││3 ││4 ││5 │││  ← 5段階評価ボタン（大きめ）
│  │  └──┘└──┘└──┘└──┘└──┘││
│  │                     ││
│  │ [ 行ったことない → ]  ││  ← スキップボタン
│  │                     ││
│  └─────────────────────┘│
│                         │
├─────────────────────────┤
│  評価  │ おすすめ │ 設定  │  ← 下部タブ（固定）
└─────────────────────────┘
```

### 6.2 操作フロー

1. 未評価の店舗が1枚ずつカードとして表示される
2. **1〜5をタップ** → 評価確定、アニメーション付きで次のカードへスライド
3. **「行ったことない」ボタン** → スキップして次のカードへ
4. 全カード完了 → まとめて `POST /submit_ratings` で送信
5. 送信完了後 → おすすめ結果画面へ遷移

### 6.3 補足仕様

- **2回目以降**: 未評価の店舗のみカードに表示（評価済み店舗は「評価済み一覧」で変更可能）
- **途中保存**: ブラウザ側で評価値を一時保持し、画面遷移やリロードでも消えないようにする（sessionStorage）
- **進捗バー**: 「3 / 20」のように何枚目かを表示し、残り枚数がわかるようにする

## 10. 管理者ダッシュボード (admin_reviews)

`/admin_reviews` で以下を表示:

### 10.1 店舗追加リクエスト管理（新規）
- pending リクエスト一覧
- 各リクエストに承認/却下ボタン
- 承認時 → objects.csv に追加、推薦再計算

### 10.2 ピボットテーブル（変更なし）
- 行: ユーザー (user_id, username)
- 列: 各店舗
- セル内容: `R: 評価値 / Z: Zスコア / RS: 推薦スコア / ★`
- 推薦スコアに応じた色分け

### 10.3 可視化 (Matplotlib → Base64 PNG)（変更なし）

| グラフ | 内容 |
|---|---|
| ユーザー類似度ヒートマップ | コサイン類似度行列を色で可視化 |
| 類似度分布ヒストグラム | 全ユーザーペアの類似度分布 (KDE付き) |
| ユーザークラスタリング | PCA(2D) + KMeans、95%信頼楕円付き散布図 |
| 推薦スコアヒストグラム | 推薦スコアの分布 (KDE付き) |

### 10.4 クラスタリングロジック（変更なし）
- クラスタ数: `max(2, ユーザー数/4 + 1)` (平均4人/クラスタ)
- PCAで2次元に次元削減 → KMeansでクラスタリング
- 95%信頼楕円をχ²分布で算出
